<objective>
Update two repository files to wire the Cloudflare Tunnel correctly:

1. `docker-compose.yml` — Change the web service host port from 8080 to 8901. The Cloudflare Tunnel routes `https://sontocodemo.sharrief.com` → `localhost:8901` on the host. The container still binds port 8080 internally, so the mapping must be `"8901:8080"` (host:container).

2. `.env.example` — Update `SITE_URL` from `http://localhost:8080` to `https://sontocodemo.sharrief.com`. This documents the canonical public URL and prompts the user to update their local `.env` so email links and self-referential redirects resolve correctly through the tunnel.

Purpose: These are the only repository-level changes required to route Cloudflare Tunnel traffic to the app. Without the port remap, the tunnel hits the wrong host port and the app is unreachable.

Output: Updated `docker-compose.yml` and `.env.example` committed to git.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host → Docker container | Port 8901 on the host is now the entry point; any process on the host can reach the container via localhost:8901 |
| Cloudflare edge → host | Cloudflare Tunnel terminates TLS and forwards to localhost:8901; Cloudflare controls what reaches the host |
| repo → developer workstations | .env.example is committed; any value here may be read by team members or public viewers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | .env.example | mitigate | Never add the tunnel token to .env.example — the plan explicitly prohibits it; the token is passed interactively at install time |
| T-02-02 | Elevation of Privilege | docker-compose.yml port 8901 | accept | Port is bound to localhost only (no 0.0.0.0 binding in this config); only the cloudflared tunnel process on the host reaches it — acceptable for local + tunnel setup |
| T-02-03 | Spoofing | SITE_URL in .env.example | accept | Value is a documentation reference; the live URL is controlled by Cloudflare DNS, not by this file |
</threat_model>
