Final docker-compose.yml content:

```yaml
services:
  web:
    build:
      context: .
      dockerfile: Dockerfile
    env_file: .env
    environment:
      # Override DB_SERVER to disable SSL for local Docker MySQL.
      # The app's src/server/lib/db.ts only enables SSL when DB_SERVER=azure or DB_SERVER=digitalocean.
      - DB_SERVER=localdreamhost
    ports:
      - "8080:8080"
    volumes:
      - .:/home/node/app
      - node_modules:/home/node/app/node_modules
      - build:/home/node/app/build
    networks:
      - sontoco-network
    depends_on:
      db:
        condition: service_healthy
    restart: unless-stopped

  db:
    build:
      context: .
      dockerfile: Dockerfile.database
    restart: unless-stopped
    env_file: .env
    ports:
      - "3306:3306"
    volumes:
      - database:/var/lib/mysql
      - ./demodata/prod:/docker-entrypoint-initdb.d
    networks:
      - sontoco-network
    environment:
      - MYSQL_ROOT_PASSWORD=${DB_ROOT_PASSWORD}
      - MYSQL_DATABASE=${DB_NAME}
      - MYSQL_USER=${DB_USER}
      - MYSQL_PASSWORD=${DB_PASSWORD}
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-u", "${DB_USER}", "--password=${DB_PASSWORD}"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s

volumes:
  node_modules:
  database:
  build:

networks:
  sontoco-network:
    driver: bridge
```

Key decisions:
- `DB_SERVER=localdreamhost` is set as an `environment:` entry on the web service. Docker Compose merges `environment:` values on top of `env_file:` values — the explicit environment entry wins. This ensures SSL is always disabled for local Docker regardless of what .env says.
- The `build:` named volume prevents the host volume mount (`.:/home/node/app`) from overlaying the webpack prod bundles that were built inside the container during `docker build`. Without this, the host's empty `build/` directory would shadow the container's `build/static/` assets and the app would 404 all static files.
- No `command:` override is needed — the Dockerfile CMD is `npm run dev:server`.
- `restart: unless-stopped` on both services is appropriate for local development (auto-restart if crashes).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env → docker compose | Secrets (DB_PASSWORD, COOKIE_SECRET) injected as env vars from .env file |
| web container → db container | MySQL connection over the sontoco-network bridge; port 3306 not auth-bypassed |
| host port 3306 → db container | MySQL port exposed to host; acceptable for local dev, restrict in production |
| host port 8080 → web container | App port exposed to host for local access and zrok tunneling |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | .env file | mitigate | .env in .gitignore; .env.example committed instead; .dockerignore excludes .env* |
| T-02-02 | Spoofing | MySQL auth | accept | MYSQL_USER/MYSQL_PASSWORD set via env_file; TypeORM connects with these credentials; no anonymous access |
| T-02-03 | Tampering | node_modules volume | accept | Named volume prevents host from tampering with container's node_modules; acceptable for local dev |
| T-02-04 | Denial of Service | MySQL not ready | mitigate | healthcheck + depends_on condition: service_healthy prevents app from starting before MySQL accepts connections |
| T-02-05 | Information Disclosure | MySQL port 3306 exposed to host | accept | Local development only; rate of exposure acceptable; do not expose in production compose |
| T-02-06 | Elevation of Privilege | DB_SERVER env var | mitigate | Explicit `environment: DB_SERVER=localdreamhost` in compose overrides env_file value, preventing accidental Azure/DO SSL profile with wrong certs |
</threat_model>
