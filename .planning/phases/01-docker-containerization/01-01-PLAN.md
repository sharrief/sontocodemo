  <action>
Rewrite Dockerfile with these specific changes:

1. Change base image from `FROM node:current` to `FROM node:20-alpine`.
   Reason: `node:current` is not reproducible across builds. Node 20 is LTS and alpine keeps the image small.

2. Add `ENV NODE_ENV=development` after the FROM line.
   Reason: nodemon/ts-node are devDependencies and must be available; setting NODE_ENV=development ensures npm install includes them. The actual runtime NODE_ENV is injected by docker-compose via env_file.

3. Keep the WORKDIR, COPY package*.json, and `RUN npm install --legacy-peer-deps` steps unchanged.

4. Keep `COPY --chown=node:node . .` and `USER node` unchanged.

5. Change `RUN npm run build` to `RUN npm run build:client`.
   Reason: `npm run build` does `rm -rf ./build && npm run build:client`. The rm -rf is fine during build, but naming it explicitly makes the intent clear. Both are equivalent — use `npm run build` (it already does the right thing: rm -rf ./build then webpack prod).
   KEEP: `RUN npm run build` — this is correct as-is.

6. Change `CMD [ "npm", "run", "start" ]` to `CMD [ "npm", "run", "dev:server" ]`.
   Reason: `npm run start` uses ts-node directly with no file watching and no nodemon restart. `npm run dev:server` uses nodemon (per nodemon.json) which wraps ts-node with watch on src/server and shared directories. This is the required hybrid runtime.

7. Keep `EXPOSE 8080` unchanged.

Final Dockerfile content:
```
FROM node:20-alpine
ENV NODE_ENV=development
RUN mkdir -p /home/node/app/node_modules && chown -R node:node /home/node/app
WORKDIR /home/node/app
COPY package*.json ./
RUN npm install --legacy-peer-deps
COPY --chown=node:node . .
USER node
RUN npm run build
EXPOSE 8080
CMD [ "npm", "run", "dev:server" ]
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env file → container | Secrets injected as env vars via docker-compose env_file |
| build context → image | Files copied into image at build time; must exclude secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Dockerfile / image layers | mitigate | `.env*` in .dockerignore prevents secrets from being baked into image layers |
| T-01-02 | Information Disclosure | .env.example | accept | .env.example contains only placeholders, safe to commit; actual .env must remain in .gitignore |
| T-01-03 | Tampering | node:current base image | mitigate | Pin to `node:20-alpine` — unpinned `node:current` can silently change behavior across builds |
| T-01-04 | Elevation of Privilege | Container user | accept | Dockerfile already runs as `USER node` (non-root); no change needed |
</threat_model>
