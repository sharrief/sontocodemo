```typescript
export const startWatchNewRequests = async () => {
  let cronJobSchedule = '0 0 0 1 0 *'; // default (disabled)
  if (env.isProduction) cronJobSchedule = '0 0 * * * *'; // hourly in prod
  if (env.isDevelopment) cronJobSchedule = '0 * * * * *'; // every minute in dev

  if (env.var.SEND_REQUEST_EMAILS) {
    const connection = await getConnection();
    if (connection) {
      const entityManager = connection.manager;
      const job = new CronJob(cronJobSchedule, () => processPendingRequests(490, entityManager));
      job.start();
      debug('Started job watchNewRequests');
    } else {
      error('Could not start job watchNewRequests because no db connection was established');
    }
  }
};
```

   `'0 0 0 * * *'` (midnight UTC daily, 6-field format with seconds). In development, fall
   back to `'0 * * * * *'` (every minute) unless `RESEED_DB_CRON` is explicitly set.

   whether running via ts-node or a compiled bundle:
   ```typescript
   const sqlFilePath = path.resolve(process.cwd(), 'demodata/prod/sontoco.sql');
   ```

6. The cron callback (`reseedDatabase`) should:
   - Read `sqlFilePath` synchronously with `fs.readFileSync(sqlFilePath, 'utf8')`
   - Split statements by `';'`, filter empty/whitespace-only strings, and prepend
     `SET FOREIGN_KEY_CHECKS=0;` as the first statement and append
     `SET FOREIGN_KEY_CHECKS=1;` as the last, so DROP TABLE + re-CREATE work without
     foreign key constraint errors
   - Execute each statement sequentially using `entityManager.query(stmt)` in a
     `try/catch`; log each error but continue so one bad statement doesn't abort the run
   - Log `info('Daily reseed complete')` when all statements are processed
   - The full callback must be `async` and handle its own `try/catch` at the outer level,
     logging any fatal error via `error(e)`

7. Obtain the TypeORM connection before scheduling: `const connection = await getConnection();`
   If null, log the error and return without starting the job.

8. Construct the CronJob and call `.start()`, then log
   `debug(\`Started job dailyReseed (schedule: ${cronSchedule})\`)`.
